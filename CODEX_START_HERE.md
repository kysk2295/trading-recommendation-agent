- live dispatcher는 기존 actionability 원장을 connector 전에 완전 재생해 동일 `(base signal ID, scan_started_at)` terminal을 다음 minute manifest에서도 connector 0건 replay로 처리한다. 새 signal ID만 새 receipt/lifecycle을 열며 2분 armed supervisor fixture는 두 READY parent와 child `1/1/0 -> 1/0/1`을 보존했다.
- query-only live evidence verifier는 completed child를 exact-minute manifest와 원래 source manifest identity, digest receipt의 bounded terminal, plan/epoch와 actionability bundle까지 대사한다. created receipt 누락·public mode·child split mismatch는 aggregate-only blocked report로 닫는다.
- actionability SQLite schema v2는 신규 artifact와 exact runtime manifest ID·snapshot 관측시각의 content-addressed creation row를 한 transaction에 append한다. live projector·dispatcher·cross-store verifier가 이 binding을 사용하며 v1 query는 무재작성이고 legacy artifact를 사후 backfill하지 않는다.
- dynamic receipt DB v2에 append-only terminal evidence를 추가했다. owner 성공은 control 3 + data 1 이상을 `BOUNDED_COMPLETE`로, final URL·control/ACK·timeout 실패는 당시 0개 이상의 receipt를 `FAILED`로 닫는다. terminal hash는 receipt IDs와 plan/epoch/time/status를 결합하고 v1 read/migration·변조를 fail-closed 재검증한다. 실제 provider smoke와 reconnect recovery는 아직 남아 있다.
- terminal store는 plan별 verified history를 UTC terminal time/epoch 순으로 재생하고 순수 reconnect policy가 최대 10회 이내 configured budget을 복원한다. complete가 있으면 새 시도를 막고, failed 수가 budget에 닿으면 차단하며, 예산이 남을 때만 exact next attempt number를 허용한다. complete 뒤 terminal이 있거나 unordered/mixed history면 fail-closed하며 policy 자체는 connector를 호출하지 않는다.
- bounded reconnect supervisor가 위 결정을 owner invocation에 연결한다. timeout·socket·handshake failure만 persisted budget 안에서 재시도하고 endpoint/protocol/ACK failure는 현재 failed terminal을 보존한 채 다음 connector 전에 차단한다. failed terminal 시각과 attempt 수가 bounded exponential eligible time을 결정하므로 restart는 남은 시간만 기다린다. wait는 stop event로 중단할 수 있고 clock regression은 connector 0건으로 차단한다. 기존 complete·budget exhaustion도 connector 0건이며 local receipt integrity 오류는 report로 숨기지 않는다. 실제 provider smoke는 아직 남아 있다.
- canonical event-bound extraction과 entity/claim corroboration read model을 추가했다. US SIP READY snapshot의 breakout·RVOL extractor는 verified dataset과 마지막 완료 분봉에 결합되어 fleet cycle의 종목별 private artifact로 연결됐다. 단일 source evidence는 unconfirmed이며 추천·승격 권한이 아니다.
- KR DART·LS normalized catalyst extraction을 추가했다. exact raw receipt link·terminal run·canonical payload와 run manifest의 keyword rules를 재생한 뒤에만 theme claim을 만들며, 서로 다른 두 provider가 같은 사전등록 theme/entity를 지지할 때만 corroborated artifact가 된다. provider·credential·계좌·주문 접근은 없다.
- US broad scanner candidate extraction을 추가했다. raw Opportunity·foundation·security-master identity와 verified candidate dataset/event를 exact 대사하고 ranking_momentum 선택 사실만 private artifact로 투영한다. source independence를 과장하지 않아 모든 claim은 unconfirmed이며 provider·broker를 열지 않는다.
- research evidence kernel이 full correction/tombstone chain의 as-of active event만 허용한다. superseded extraction은 차단되고 correction은 새 event-bound extraction이 필요하며 미래 correction은 조기 반영되지 않는다. 과거 artifact는 immutable로 유지된다.
- KIS scan/watch의 기존 research projection opt-in이 scanner commit 직후 evidence artifact를 자동 생성한다. store 부모의 deterministic `research-evidence/`에 content-addressed artifact를 쓰며 exact retry는 하나만 유지하고 별도 provider·broker 호출은 없다.
- Alpaca SIP trade `t/c/x` raw-first vertical 위에 exact SIP endpoint, connected/auth/trade subscription control receipt와 epoch별 data owner를 연결했다. mode-600 append-only stream audit가 control hash·exact ACK·sequence·terminal record를 재검증하며 유효 frame 0건·protocol failure는 `failed`로 닫는다. 데이터 뒤 disconnect와 새 epoch reconnect를 local fixture로 재현하고, 각 receipt의 단일 epoch 소유권과 수신시간을 검증한다. clean 단일 epoch만 마지막 raw frame 수신시각까지 bounded continuity를 attested하며 epoch 사이 provider backfill 증거가 없으면 correction/cancel을 모두 받아도 complete history로 승격하지 않는다. 실제 provider 연결 smoke·장기 soak는 아직 없고 기존 REST minute-bar capability는 `snapshot_only`다.
- 명시적 arm·current NYSE regular session·private Alpaca market-data credential 뒤에만 위 trade stream을 여는 bounded smoke CLI를 추가했다. frame/timeout은 10 이하이고 매 frame 뒤 session/date를 다시 확인하며 한 epoch의 raw audit→canonical Parquet→complete coverage→private report를 확정한다. 휴장 fixture는 credential·network·state 전에 차단됐고 실제 provider 연결은 아직 0건이다.
- SIP stream schema v2는 session 준비 전 handshake·endpoint·control·provider rejection을 별도 failed connection attempt로 보존한다. 공식 402·406·409만 auth failure·connection limit·insufficient subscription으로 분류하고 나머지는 generic provider/protocol/transport code로 축소한다. exception text와 credential은 attempt row에 없고 기존 schema v1은 다음 Writer에서만 table·trigger를 추가한다. local fixture CLI는 connection-limit과 handshake 실패를 network 0건으로 재현한다.
- bounded reconnect supervisor는 symbol/date별 durable connection budget 3회를 attempt+terminal 원장에서 재생한다. transport·handshake·provider 500·준비 후 disconnect만 1초·2초 backoff로 재시도하며 402·406·409·endpoint·protocol은 no-retry다. 재시작은 budget을 초기화하지 않고 bounded complete가 있으면 operation 0회이며, 실패 뒤 성공은 continuity를 주장하지 않는다. 명시적 run ID별 lifecycle은 별도 mode-600 append-only hash-chain 감사 원장에 기록하고 연결 또는 backoff 전 shutdown은 terminal stopped evidence로 닫는다. 실제 장중 soak와 production smoke 결합은 아직 없다.
